# Contributing to Aperio

Thanks for your interest in contributing. Aperio is early-stage and moving fast — every contribution helps.

## Ways to contribute

- 🐛 **Report bugs** — open an issue with steps to reproduce, your OS, model, and terminal output
- 💡 **Suggest features** — open an issue describing the problem you're trying to solve
- 🔧 **Submit a fix** — fork, fix, PR with a clear description of what changed and why
- 🧪 **Test on your hardware** — run the community testing issue and leave a comment with your results
- 📖 **Improve docs** — fix typos, clarify setup steps, add examples

## Before opening a PR

- Keep changes focused — one fix or feature per PR
- Test your changes locally with at least one model
- Update the README if your change affects setup or behavior
- Use clear commit messages — `feat:`, `fix:`, `docs:`, `chore:` prefixes

## Questions?

Open an issue or start a discussion. There are no stupid questions here.
